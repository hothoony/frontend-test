<table>
  <tr>
    <td rowspan="4">이미지</td>
    <td rowspan="2">2025</td>
    <td>12월</td>
    <td>내용</td>
  </tr>
  <tr>
    <td>11월</td>
    <td>내용</td>
  </tr>
</table>

✅ 장점

  • 간단하고 직관적
  • 모든 브라우저 지원
  • rowspan으로 셀 병합 쉬움

❌ 단점

  • 의미적으로 잘못된 사용
  • 접근성 문제
  • 반응형 어려움
  • 스타일링 제한적

.timeline {
  display: grid;
  grid-template-columns: 100px 80px 80px 1fr;
}
.image-cell { grid-row: span 4; }
.year-2025 { grid-row: span 2; }
.year-2024 { grid-row: span 2; }

✅ 장점

  • 완벽한 셀 병합
  • 테이블과 같은 결과
  • 의미적으로 올바름
  • 완전한 반응형
  • 접근성 우수

❌ 단점

  • IE 미지원 (IE11 부분지원)
  • 복잡한 구조에선 학습 필요

🏆 결론

현재 권장사항:

  1. 데이터 테이블이라면: 여전히 HTML <table> 사용 (의미적으로 올바름)
  2. 레이아웃 목적이라면: CSS Grid (grid-row: span 사용)
  3. 구형 브라우저 지원 필요시: Flexbox
  4. 매우 간단하다면: 테이블도 나쁘지 않음
